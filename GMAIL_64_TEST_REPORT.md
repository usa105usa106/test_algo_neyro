## Проверено

1. Стандартный Coolify magic URL `SERVICE_URL_GMAIL-AUTH` нормализуется в HTTPS.
2. Callback формируется как `/gmail/callback`, health URL — как `/healthz`.
3. Callback-сервер запускается даже без публичного URL и отдаёт локальный `/healthz`.
4. Реальный локальный listener успешно запущен на порту `80`; запрос вернул HTTP 200 и `ok=true`.
5. Одноразовая Telegram-кнопка health probe подтверждает, что внешний запрос дошёл до процесса бота.
6. OAuth URL и ввод Client ID/Secret блокируются до подтверждения probe.
7. Истёкший probe не разблокирует Gmail-настройку.
8. Client ID и Client Secret хранятся зашифрованно.
9. При повторном открытии хранилища Storage ID сохраняется, boot count увеличивается, Gmail client остаётся доступен.
10. Gmail bundle зеркалируется в резервный volume.
11. Если основное хранилище пустое или успело создать новый ключ, Gmail client восстанавливается из резервного volume и пере-шифровывается под основной ключ.
12. При таком восстановлении существующие MEXC API credentials не повреждаются.
13. Gmail OAuth token и журнал отправленных архивов используют постоянное хранилище.
14. Один ZIP отправляется в Gmail только после успешной отправки в Telegram.
15. Второй вызов для того же имени/размера/SHA-256 блокируется.
16. Параллельные вызовы не создают два письма.
17. Несовпадение имени Telegram-вложения блокирует Gmail.
18. Изменение содержимого ZIP после Telegram блокирует Gmail.
19. При неизвестном результате сетевой отправки автоповтор блокируется во избежание дубля.
20. Тема письма и имя вложения содержат точное имя исходного ZIP.
21. Docker Compose корректно разбирается как YAML.
22. Compose слушает `80`, не содержит gateway и использует два постоянных хранилища.
23. Все Python-файлы проходят `py_compile`.

## Ограничение проверки

В этой среде невозможно развернуть именно пользовательский reverse proxy Coolify. Поэтому внешний маршрут окончательно подтверждается встроенной кнопкой Telegram: бот не разрешит ввод Client ID/Secret, пока браузерный probe реально не попадёт в процесс бота. В отличие от прошлых версий это не обещание «должно работать», а обязательная проверка до ввода секретов.
