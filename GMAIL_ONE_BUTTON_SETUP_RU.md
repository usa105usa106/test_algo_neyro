# Gmail OAuth — версия 65

Версия 65 убирает ручную работу с `/healthz`, портом `8080` и полем Domains в Coolify.

## Что делает бот сам

- callback-сервер слушает стандартный внутренний порт `80`;
- Coolify создаёт wildcard-URL через `SERVICE_URL_GMAIL-AUTH_80`;
- бот показывает готовый Redirect URI `/gmail/callback`;
- бот показывает кнопку проверки сервера, а не заставляет вручную менять окончание ссылки;
- ввод Client ID/Secret блокируется, пока публичный запрос реально не достиг бота;
- Client ID, Client Secret, OAuth token и журнал отправленных ZIP сохраняются одновременно в двух хранилищах.

## Порядок подключения

1. Нажать `📧 Подключить Gmail`.
2. Нажать `🌐 1. Открыть проверку сервера`.
3. В браузере должен появиться JSON:

```json
{"ok": true, "service": "gmail-oauth-callback", "probe_confirmed": true}
```

4. Вернуться в Telegram и нажать `✅ 2. Проверить результат`.
5. Только после сообщения `Публичный callback реально достиг бота` появится кнопка ввода Client ID/Secret.
6. В Google OAuth Client вставить Redirect URI, который показал бот.
7. Отправить боту Client ID, затем Client Secret.
8. Нажать `🔐 Войти через Google`.
9. После подключения нажать `🧪 Отправить тест` и проверить Gmail → Отправленные.

## Порт и домен

Вручную указывать `:8080`, `/healthz` или Domains не нужно.

Контейнер слушает:

```text
0.0.0.0:80
```

Coolify-маршрут создаётся переменной:

```text
SERVICE_URL_GMAIL-AUTH_80
```

Google получает только HTTPS callback:

```text
https://<домен-coolify>/gmail/callback
```

## Сохранение данных

Основное хранилище VPS:

```text
/data/chatgpt-scan-bot-storage
```

Резервное хранилище Docker:

```text
chatgpt_scan_storage
```

В основной папке находятся:

```text
state/fernet.key
secrets/gmail_client.enc.json
secrets/gmail_oauth.enc.json
state/gmail_sent_archives.json
```

В резервном volume бот атомарно хранит один согласованный файл:

```text
gmail_bundle_backup.json
```

Он содержит ключ и соответствующие ему зашифрованные Gmail-файлы одной генерации. Если основная папка после обновления пустая, бот восстанавливает Gmail-настройки из этого bundle. Если в основной папке уже есть другой ключ для MEXC API, Gmail-данные пере-шифровываются под него — MEXC-настройки не ломаются.

В сообщении Gmail бот показывает `Хранилище: <ID>, запусков: <число>`. После Redeploy ID должен остаться тем же, а число запусков увеличиться.

## Отправка архивов

- сначала ZIP успешно отправляется в Telegram;
- затем Gmail получает тот же логический ZIP с тем же именем;
- перед письмом повторно проверяются имя, размер и SHA-256;
- один и тот же ZIP повторно не отправляется;
- письмо сохраняется в Gmail → Отправленные;
- тема содержит режим и точное имя архива.
