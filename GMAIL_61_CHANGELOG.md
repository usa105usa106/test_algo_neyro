# v61_full_GMAIL_TELEGRAM_SETUP_SENT_DEDUP

- Google Client ID и Client Secret теперь вводятся через Telegram, а не через Coolify.
- Чувствительные сообщения сразу удаляются, если Telegram разрешает удаление.
- OAuth-клиент хранится зашифрованно в persistent volume.
- Смена OAuth-клиента удаляет старый Gmail refresh token, чтобы не смешивать разные клиенты.
- ZIP отправляется в Gmail только после успешной отправки в Telegram.
- Перед Telegram фиксируются имя, размер и SHA-256; перед Gmail они проверяются повторно.
- Имя вложения и имя файла, подтверждённое Telegram, обязаны совпасть.
- Добавлен постоянный журнал защиты от дублей по Gmail-адресу + имени + размеру + SHA-256.
- При неопределённом сетевом результате автоповтор блокируется, чтобы не отправить два письма.
- Подтверждение указывает папку `Gmail → Отправленные`.
- Task-файлы, сборщики архивов, Intraday engine, MEXC downloader и остальные режимы не изменялись.
